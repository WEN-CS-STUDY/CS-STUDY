싱글톤 패턴은 객체 지향 프로그래밍에서 특정 클래스가 단 하나의 인스턴스만을 생성하여 사용하기 위한 패턴이다.

따라서 여러 번 호출하여도 생성되어있는 하나의 인스턴스만 접근가능하다.

```jsx
public class Singleton {
    // 정적 참조 변수(싱글톤 객체를 담을 변수)
    private static Singleton singletonObject;

    // private 생성자
    private Singleton() {
    }
    
    // getInstance()
    public static Singleton getInstance() {
        if (singletonObject == null) {
            singletonObject = new Singleton();
        }
        
        return singletonObject;
    }
}
```

코드를 보면 생성자를 private 을 사용하여 접근을 못하게 막고, getInstance 를 통해 객체가 존재하면 그 객체를 보내주고 없으면 생성해서 보내주기 때문에 하나의 인스턴스만 존재하게 만든다.

사용 예시

커넥션 풀 / 스레드 풀

두 자원 풀 모두 미리 여러개의 커넥션, 스레드 들을 만들어 생성과 삭제의 오버헤드를 줄이기 위하여 재사용을 한다.

이때 커넥션 풀, 스레드 풀 자체를 싱글톤으로 생성하여 서버 전체에서 동일한 풀을 사용하고,  풀을 생성할때의 오버헤드도 한번만 실행되도록 할 수 있어 싱글톤을 사용한다.

우리가 사용하는 Spring 의 Bean 또한 싱글톤으로 사용된다.

우리가 어노테이션을 통해 여러개의 Bean 을 저장하고, 스프링에서는 이 Bean 들의 라이프 사이클을 관리하면서 우리가 사용할때마다 DI 를 통해 의존성을 주입시켜준다. 이때 주입해주는 객체는 단 한개의 인스턴스를 스프링 컨테이너가 가지고 있고, 호출될때 마다 이 한개의 인스턴스를 제공한다.
