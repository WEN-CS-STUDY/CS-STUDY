NoSQL 은 비관계형 데이터베이스로 기존의 관계형 데이터베이스의 한계를 극복하기 위한 데이터 저장소의 새로운 형태이다.

NoSQL 은 기본적으로 읽기와 쓰기에 대한 효율이 높다.

이에 반대로 관계형 데이터베이스의 특성인 데이터의 일관성과 ACID 특성은 모두 가지고 잊지 않는다.

NoSQL 의 종류

- Key - Value Store

Key 를 이용해 value 에 접근하는 구조로, 저장과 조회에 충실한 데이터베이스

value 에는 숫자나 문자열 등 아무 값(리스트, JSON)이나 가능하다.

따라서 RDB와 다르게 고정 스키마가 없고, 해시 테이블을 기반으로 사용하여 O(1)에 가까운 빠른 조회가 가능하다.

자주 사용되는 예시로는

1. 성능 향상을 위해 관계형 데이터베이스에서 데이터 캐싱

2. 장바구니 같은 웹 애플리케이션에서 일시적인 속성 추적

3. 모바일 애플리케이션용 사용자 데이터 정보와 구성 정보 저장

4. 이미지나 오디오 파일 같은 대용량 객체 저장

대표 DB 로는 Redis, Riak, AWS DynamoDB 등이 있다.

- Document Store

데이터를 Document 형태로 저장하는 방식

즉 JSON 이나 XML 과 같은 표준 형식으로 저장을 한다는 뜻이다.

위와 마찬가지로 고정 스키마가 없다.

즉 예를들어 

{
_id: 1,
name: "승우",
age: 21
}

{
_id: 2,
name: "철수",
email: "[chulsoo@example.com](mailto:chulsoo@example.com)",
address: { city: "Seoul" }
}

이런 식으로 다른 필드를 저장할 수 있고, 이부분에 대한 관리는 에플리케이션 단계에서 이루어져야 한다.

자주 사용되는 예시로는

1. 대용량 데이터를 읽고 쓰는 웹 사이트용 백엔드 지원

2. 제품처럼 다양한 속성이 있는 데이터 관리

3. 다양한 유형의 메타데이터 추적

4. JSON 데이터 구조를 사용하는 애플리케이션

5. 비정규화된 중첩 구조의 데이터를 사용하는 애플리케이션

대표 DB로는 MongoDB, CouchDB, Couchbase 등이 있다.

- Column Family Store

Column과 Row 를 통해 스키마를 정의한다.

이때 컬럼 수가 많다면 관련된 컬럼들을 컬렉션으로 묶을 수 있다.

이때 묶인 컬럼들을 Column Family 라고 한다.

Document Database와 마찬가지로 미리 정의된 스키마를 사용하지 않으므로 개발자가 데이터를 입력하는 시점에 원하는대로 컬럼을 추가할 수 있다.

![image.png](/ASSET/NoSQL.png)

테이블간 조인을 지원하지 않는다.

예를들어 우리는 정규화를 통해 테이블의 정보들을 중복을 최소화 하기위해 여러개의 테이블로 쪼개어 저장하지만 이 Column Family Database 는 일반적으로 비정규화 되어 있으며, 한 객체에 관련된 정보를 여러개의 컬럼을 통해 하나의 단일 Row 에 넣어서 보관한다.

자주 사용하는 예시로는

1. 데이터베이스에 쓰기 작업이 많은 애플리케이션

2. 지리적으로 여러 데이터 센터에 분산되어 있는 애플리케이션

3. 복제본 데이터가 단기적으로 불일치하더라도 큰 문제가 없는 애플리케이션

4. 동적 필드를 처리하는 애플리케이션

5. 수백만 테라바이트 정도의 대용량 데이터를 처리할 수 있는 애플리케이션

대표 DB로는 Hbase, Cassandra(facebook), GCP BigTable(google)

- Graph Store

노드 사이의 관계를 엣지로 표현하는 그래프 구조를 사용해 데이터를 표현하고 저장하는 데이터베이스

예를들어 sns 의 친구관계를 추적할때 RDB 를 사용한다면 조인을 통해 나의 친구의 친구 등을 찾는등 연산의 크기가 크지만, Graph database 의 경우 나의 노드에서 친구인 엣지를 찾아가기만 하면 쉽게 나와 친구인 사람, 친구의 친구인 사람등을 찾을 수 있다.

따라서 데이터 간의 관계가 중요한 애플리케이션에 최적.

또한 새로운 관계나 속성을 추가하는 경우도 단지 노드 만 연결해주면 되는 부분이라 확장성과 유연성도 높다.

자주 사용되는 예시

1. SNS 관계 탐색
2. 추천 시스템 (상품 → 카테고리 → 비슷한 상품)
3. 네트워크 / 경로 탐색

대표 DB로는 Neo4j, Amazon Neptune 등
