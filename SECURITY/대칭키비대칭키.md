### 대칭키 암호화 방식

- 암호화와 복호화에 사용되는 키가 동일한 암호화 방식
- 대칭키를 가진 사람은 암호화와 복호화를 할 수 있음.

### 공개키 암호화 방식

- 키 배송 문제를 극복하기 위해 등장.
- 암호화와 복호화에 사용되는 키가 다른 암호화 방식
- 공개키와 개인키를 사용 (공개키로 암호화한 것은 오직 개인키로만 복호화 가능함).
- 동작 과정
    - 나에게 암호문을 전달하고 싶은 대상에게 공개키를 전달한다.
    - 공개키를 받은 사람은 공개키로 암호화하고, 암호문을 보낸다.
    - 나는 암호문을 받아 개인키로 복호화한다.
    

### 사용 사례

- 대칭키 암호화 알고리즘 : DES, 3DES, AES, SEED 등
- 공개키 암호화 알고리즘 : RSA, DSA

### 장점

- 대칭키 알고리즘
    - 공개키 암호화에 비해 속도가 빠르다.
- 공개키 알고리즘
    - 키 배송 문제를 해결할 수 있음. (안전성)

### 한계 및 해결방안

- 대칭키 알고리즘
    - 키 배송 문제가 발생함 (키를 어떻게 전달해?)
    - 사람이 증가할수록 관리해야 할 키가 증가함.
    - 이를 해결하기 위해 사전 공유 및 공개키 암호화 방식 사용등이 있음.
- 공개키 알고리즘
    - 속도가 느리다.

### 전자서명

- 무결성과 인증을 보장
- 공개키 암호화 방식 사용
- 동작 방식
    - 메시지를 보낼 사용자는 메시지에 고정된 해시 함수를 적용한다 (SHA-256)
    - 그 값을 메시지를 보내는 송신자의 비밀키로 서명한다.
    - 수신자는 메시지를 동일한 해시 함수에 적용한다.
    - 또한 2번에서 받은 값을 송신자의 공개키로 복호화한다.
    - 3번과 4번이 일치하면 인증 완료, 아니면 위조된 것

### 전자서명의 위변조

- 위조 : 당연히 다른 사람의 비밀키로 암호화한게 오면 해시값이 달라진다.
- 변조 : 메시지가 조금이라도 달라지면 해시 함수를 적용할 때 이상한 해시 값이 나올 것이다.
