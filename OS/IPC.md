### IPC

- 하나의 컴퓨터 안에서 실행되는 프로세스 간에 발생하는 통신
- 기본적으로 멀티 프로세스에서 프로세스는 각자 독립적인 주소 공간을 가지고 있어 서로간의 통신이 불가능하지만 IPC를 사용하면 가능하다.
- 공유 메모리, 소켓, 파이프, 메시지 큐, 세마 포어 등이 있음.

### 공유 메모리

![image.png](/ASSET/공유메모리.png)

- 여러 프로세스가 서로 통신할 수 있도록 메모리를 공유하는 것
- 메모리 자체를 공유하기 때문에 오버헤드가 발생하지 않기 때문에 가장 빠르다.
- 같은 메모리 영역을 여러 프로세스가 공유하기 때문에 읽고 쓸때 동기화가 필요하다.

### 소켓

![image.png](/ASSET/소켓.png)

- 네트워크 인터페이스 (TCP, UDP, HTTP)을 기반으로 통신하는 것
- 서버 - 클라이언트 구조를 가진다.
- 각각의 다른 컴퓨터의 프로세스간 데이터를 교환하기 위해 사용된다.
- 프로세스간 양방향 통신이 가능하다.

### 파이프

![image.png](/ASSET/파이프.png)

- 부모 자식 프로세스간 통신을 위해 사용
    - 한 쪽은 쓰고 파이프를 통해 전달하고 한 쪽은 읽는 형태
- 파이프 내에 데이터 용량도 제한되어있음.
- 쓰기 프로세스가 읽기 프로세스보다 더 빠르게 데이터를 처리할 수 없음.

### 익명 파이프

![image.png](/ASSET/익명파이프.png)

- 프로세스 사이의 FIFO 기반의 통신 채널을 만들어 통신하는 것

### 네임드 파이프

![image.png](/ASSET/네임드파이프.png)

- 다른 네트워크 상에서도 통신할 수 있는 파이프

### 메시지 큐

![image.png](/ASSET/메시지큐.png)

- 메시지를 큐 자료구조 형태로 관리하는 버퍼를 만들어 통신하는 것
- 메시지 지향 미들웨어를 구현한 시스템을 구현한 시스템
    - 프로세스가 메시지를 보내거나 받기 전 큐를 초기화
    - 보내는 프로세스의 메시지는 큐에 복사되어 받는 프로세스에 전달
