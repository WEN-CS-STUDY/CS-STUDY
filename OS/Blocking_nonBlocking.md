### Blocking vs NonBlocking (제어권의 유무)

- 블로킹
    - 자신의 작업을 진행하다가 다른 주체의 작업이 시작되면 다른 작업이 끝날 때까지 기다렸다가 자신의 작업을 시작함.
    - 직원이 서류를 상사에게 넘김, 상사가 서류를 다 볼때까지 직원에게 기다리라고함, 다 봤으니 돌아가라고 함.
- 논블로킹
    - 다른 주체의 작업에 관련없이 자신의 작업을 하는 것
    - 병렬 실행과 관련이 깊음.
    - 직원이 서류를 상사에게 넘김, 상사는 읽어볼테니 돌아가라고 얘기함, 상사는 그 사이에 서류를 읽고, 직원은 자기의 할일을 함.

### sync vs Async (작업의 순차적 실행 여부)

- 동기
    - 작업을 동시에 수행하거나, 동시에 끝나거나, 끝나는 동시에 시작함을 의미
    - 요청한 작업에 대해 순서가 지켜지는 것.
    - 직원이 서류를 상사에게 넘김, 상사는 아무 얘기도 안하고 읽음 (다른 일을 해도 신경쓰지 않음), 직원이 결과를 전달 받으면 그걸 받고 바로 처리함.
- 비동기
    - 시작, 종료가 일치하지 않고, 끝나는 동시에 시작을 하지 않음.
    - 요청한 작업에 대해 순서가 지켜지지 않아도 됨.
    - 직원이 서류를 상사에게 넘김, 기다리거나 다른일을 해도 신경 안쓰고 일을 바로 처리하지 않아도 됨, 상사가 서류를 다 읽고 직원에게 보내줌, 직원은 언젠가는 이 일을 처리하면 됨.

### Blocking Sync

- 다른일에게 제어권을 넘겨주고 그것을 처리하고 받으면 이어서 작업을 한다.
- 다른 작업의 결과가 자신의 작업에 영향을 주는 경우에 활용.
- 작업량이 많으면 시간이 오래 걸릴 수 있다는 한계가 있음.
- ex. 프로그램의 입출력

### Non-Blocking Sync

- 다른일에게 제어권을 넘겨주지 않고, 중간에 계속 작업이 끝났는지 물어보고, 작업이 끝났으면 이를 받아 이어서 작업한다.
- ex. 게임에서 맵을 넘어갈 때, (로드율이 얼마인지 보여주고 싶을 때), Java 언어 (쓰레드 만들어서 처리, 메인에서는 해당 쓰레드가 작업을 완료했는지 여부를 계속해서 확인한다.)

### Blocking Async

- 다른일에게 제어권을 넘겨주고, 결과를 바로 처리하지 않아도 됨. (자신의 일이 끝나고 처리해도 됨.)
- 굳이 이 방법을 쓰지는 않는다.

### Non-Blocking Async

- 다른일에게 제어권을 넘겨주지 않고, 결과를 바로 처리하지 않아도 됨. (자신의 일이 끝나고 처리해도 됨)
- 다른 작업의 결과가 자신의 작업에 영향을 주지 않은 경우에 활용
- ex. 자바스크립트의 API 요청을 하고 콜백으로 추가 작업 처리를 할 때
